## 方式一：IDEA 启动（推荐）⭐

### 📌 步骤 1：导入项目到 IDEA

1. 打开 IntelliJ IDEA
2. 点击 **File → Open**
3. 选择项目根目录 `D:\project\springboot3wd7d5i4`
4. 等待 Maven 自动导入依赖（右下角有进度条）

### 📌 步骤 2：配置数据库

1. 确保 MySQL 已启动
2. 创建数据库：
```sql
CREATE DATABASE springboot3wd7d5i4 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

3. 导入数据：
```bash
mysql -u root -p springboot3wd7d5i4 < db/springboot3wd7d5i4.sql
```

4. 修改配置文件 `src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/springboot3wd7d5i4?...
    username: root        # 改成你的用户名
    password: 123456      # 改成你的密码
```

### 📌 步骤 3：启动后端

1. 找到主类：`src/main/java/com/SpringbootSchemaApplication.java`
2. 右键点击文件
3. 选择 **Run 'SpringbootSchemaApplication'**
4. 等待启动完成，看到以下日志表示成功：
```
Started SpringbootSchemaApplication in X.XXX seconds
```

### 📌 步骤 4：启动前端

#### 方法 A：使用 IDEA 内置终端

1. 在 IDEA 底部点击 **Terminal**
2. 进入前端目录：
```bash
cd src/main/resources/admin/admin
```

3. 安装依赖（首次运行）：
```bash
npm install
```

4. 启动开发服务器：
```bash
npm run serve
```

#### 方法 B：使用外部命令行

1. 打开 PowerShell 或 CMD
2. 进入前端目录：
```bash
cd D:\project\springboot3wd7d5i4\src\main\resources\admin\admin
```

3. 安装依赖（首次运行）：
```bash
npm install
```

4. 启动开发服务器：
```bash

```

## 常见问题解决

### ❌ 问题 1：端口被占用

**错误信息**：
```
Port 8080 was already in use
```

**解决方案**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080

# 终止进程（替换 PID）
taskkill /F /PID <PID>
```

或修改 `application.yml` 中的端口：
```yaml
server:
  port: 8081
```

### ❌ 问题 2：数据库连接失败

**错误信息**：
```
Could not open JDBC Connection for transaction
```

**解决方案**：
1. 确认 MySQL 服务已启动
2. 检查数据库名、用户名、密码是否正确
3. 确认防火墙未阻止 3306 端口

### ❌ 问题 3：npm install 失败

**错误信息**：
```
npm ERR! code ECONNRESET
```

**解决方案**：
```bash
# 切换淘宝镜像
npm config set registry https://registry.npmmirror.com

# 清理缓存后重试
npm cache clean --force
npm install
```

### ❌ 问题 4：前端页面空白

**解决方案**：
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签的错误信息
3. 确认后端 API 是否正常启动
4. 检查 `vue.config.js` 中的代理配置

### ❌ 问题 5：Maven 命令找不到

**错误信息**：
```
mvn : 无法将"mvn"项识别为 cmdlet
```

**解决方案**：
1. 下载 Maven：https://maven.apache.org/download.cgi
2. 解压到目录（如 `C:\Program Files\Maven`）
3. 添加环境变量：
   - 新建 `MAVEN_HOME` = `C:\Program Files\Maven`
   - 编辑 `Path`，添加 `%MAVEN_HOME%\bin`
4. 重启 PowerShell 验证：`mvn -version`

## GitHub CI/CD 配置详解

### 🎯 什么是 CI/CD？

- **CI（持续集成）**：代码提交后自动构建、测试
- **CD（持续部署）**：通过测试后自动部署到服务器

### 📂 项目中的 CI/CD 文件

位置：`.github/workflows/maven.yml`

### 🔧 配置步骤

#### 步骤 1：创建 GitHub 仓库

```bash
# 初始化 Git
git init

# 添加远程仓库
git remote add origin https://github.com/你的用户名/springboot-paper-topic.git

# 提交代码
git add .
git commit -m "Initial commit"
git push -u origin main
```

#### 步骤 2：配置 GitHub Secrets

在 GitHub 仓库中设置敏感信息：

1. 进入仓库 → **Settings** → **Secrets and variables** → **Actions**
2. 添加以下 Secrets：

| Secret 名称 | 值 | 说明 |
|------------|-----|------|
| `DOCKER_USERNAME` | 你的 Docker Hub 用户名 | 推送镜像用 |
| `DOCKER_PASSWORD` | 你的 Docker Hub 密码 | 推送镜像用 |
| `DEPLOY_HOST` | 服务器 IP 地址 | 生产服务器 |
| `DEPLOY_USER` | 服务器用户名 | SSH 登录用 |
| `DEPLOY_SSH_KEY` | SSH 私钥 | 免密登录 |

#### 步骤 3：创建 Dockerfile

在项目根目录创建 `Dockerfile`：

```dockerfile
FROM openjdk:8-jdk-alpine

WORKDIR /app

COPY target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 步骤 4：创建 docker-compose.yml

```yaml
version: '3.8'

services:
  app:
    image: ghcr.io/你的用户名/springboot-paper-topic:latest
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=mysql
      - DB_USERNAME=root
      - DB_PASSWORD=${DB_PASSWORD}
    depends_on:
      - mysql
  
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}
      MYSQL_DATABASE: springboot3wd7d5i4
    volumes:
      - mysql-data:/var/lib/mysql
      - ./db:/docker-entrypoint-initdb.d

volumes:
  mysql-data:
```

#### 步骤 5：触发 CI/CD

每次执行以下操作时会自动触发：

```bash
# 推送到 main 分支
git push origin main

# 创建 Pull Request
# 在 GitHub 上创建 PR 到 main 分支
```

### 📊 CI/CD 流程说明

```
┌─────────────┐
│  代码推送    │
└──────┬──────┘
       ↓
┌─────────────────┐
│ Job 1: Build    │
│ - 检出代码       │
│ - 设置 JDK 8     │
│ - Maven 编译     │
│ - 运行单元测试   │
│ - 生成覆盖率报告 │
└──────┬──────────┘
       ↓
┌─────────────────┐
│ Job 2: Security │
│ - OWASP 扫描    │
│ - 依赖安全检查   │
└──────┬──────────┘
       ↓
┌──────────────────┐
│ Job 3: Deploy    │
│ - 构建 Docker 镜像│
│ - 推送到 GHCR    │
│ - SSH 部署到服务器│
└──────────────────┘
```

### 🔍 查看 CI/CD 状态

1. 进入 GitHub 仓库
2. 点击 **Actions** 标签
3. 查看最近的工作流运行状态
4. 点击具体任务查看详细日志

### 💡 优化建议

1. **缓存 Maven 依赖**：加速构建
```yaml
- uses: actions/cache@v3
  with:
    path: ~/.m2
    key: ${{ runner.os }}-m2-${{ hashFiles('**/pom.xml') }}
```

2. **并行执行任务**：节省时间
```yaml
jobs:
  build:
    runs-on: ubuntu-latest
  test:
    runs-on: ubuntu-latest
    needs: build
```

3. **条件触发部署**：仅 main 分支部署
```yaml
if: github.ref == 'refs/heads/main'
```
