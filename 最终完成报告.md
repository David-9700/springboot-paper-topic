## 🔧 可选：配置部署 Secrets

如果您有生产服务器并想要自动部署，需要配置以下 Secrets：

### 访问路径
```
https://github.com/David-9700/springboot-paper-topic/settings/secrets/actions
```

### 必需的 Secrets

| Secret 名称 | 值 | 说明 |
|------------|-----|------|
| `DEPLOY_HOST` | 服务器 IP | 例如：192.168.1.100 |
| `DEPLOY_USER` | SSH 用户名 | 例如：root |
| `DEPLOY_SSH_KEY` | SSH 私钥 | 完整的私钥内容 |

### 如果没有服务器

**不用担心！** 前两个阶段（Build & Test、Security Scan）仍然会正常运行，只是不会执行部署。您可以：

1. 在本地测试 Docker 构建
2. 手动下载 Docker 镜像
3. 以后需要时再配置部署

## 📊 npm install 进度

### 当前状态：⏳ 仍在安装中

**预计剩余时间**：**2-5 分钟**

**原因**：
- 首次安装需要下载 980+ 个 npm 包
- 包含 Vue.js、Element UI、ECharts 等大型库
- 网络速度影响下载时间

### 加速方案（如果太慢）

如果等待超过 10 分钟，可以：

```powershell
# 1. 停止当前安装（Ctrl+C）

# 2. 切换到淘宝镜像
npm config set registry https://registry.npmmirror.com

# 3. 清理缓存
npm cache clean --force

# 4. 重新安装
cd D:\project\springboot3wd7d5i4\src\main\resources\admin\admin
npm install
```

## 💡 常见问题

### Q1: Actions 页面没有显示工作流？

**解决**：
- 刷新页面
- 等待 1-2 分钟
- 检查 `.github/workflows/maven.yml` 是否存在

### Q2: CI/CD 失败了怎么办？

**解决**：
- 查看详细日志
- 检查是否是测试失败
- 本地运行 `mvn test` 验证

### Q3: npm install 一直卡住？

**解决**：
- 使用淘宝镜像（见上方加速方案）
- 或者等待更长时间（首次安装可能很慢）
