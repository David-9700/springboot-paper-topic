## 💡 相关知识

### 为什么会出现构造函数歧义？

`PageUtils` 有多个构造函数：
```java
public PageUtils(com.baomidou.mybatisplus.plugins.Page<?> page)
public PageUtils(Map<String, Object> map)
```

当传入 `null` 时，编译器无法确定应该调用哪个构造函数。

### 解决方案

1. **明确类型** - 使用具体的类而不是 null
2. **类型转换** - `(Page<?>)null` 或 `(Map)null`
3. **替代方案** - 使用 HashMap（我们选择的方案）✅

### Mockito ArgumentMatchers

使用 Mockito 时，必须导入所有使用的 matchers：

```java
import static org.mockito.ArgumentMatchers.*;

// 常用的 matchers
anyLong()      // 匹配任意 long
anyString()    // 匹配任意 String
anyMap()       // 匹配任意 Map
any()          // 匹配任意对象
eq(value)      // 匹配特定值
```
